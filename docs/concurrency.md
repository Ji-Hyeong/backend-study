# Concurrency Lab

## 학습 질문

- 낙관적 락과 비관적 락은 어떤 충돌 모델에서 선택하는가?
- Redis 분산락은 DB 트랜잭션과 어떤 순서로 잡고 풀어야 하는가?
- 멱등성 키는 중복 요청과 동시 요청 중 어디까지 방어하는가?
- 포인트, 재고, 쿠폰 수량 차감은 왜 단순 `find -> update`로 부족한가?

## 실험 계획

- 같은 차감 API를 무락, 낙관적 락, 비관적 락, Redis lock 버전으로 구현한다.
- `CountDownLatch` 기반 테스트로 실패를 재현한다.
- 실패한 요청 수, 최종 잔액, 원장 수를 함께 검증한다.
